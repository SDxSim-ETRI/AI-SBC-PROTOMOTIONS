collect_data_suit.py 로 뽑은 zarr 데이터를 순수 코드 리딩으로 검증한 결과입니다. 핵심 결론: 물리 상태 배열은 모두 올바르게 정렬되어 정상이며, 토크 라벨도 준비되어 있습니다. 단, 케이블·고관절 파생 헬퍼 필드의 하드코딩 인덱스는 27-DOF suit 모델 전용이라는 점만 주의하면 됩니다.

06

바로 쓰는 로더 — 인덱스 자동 검출

copy & run

다른 머신에서 그대로 실행. 하드코딩 없이 dof_names 로부터 고관절 인덱스를 유도하므로 23/27/31-DOF 어느 모델이든 안전합니다.

import zarr, numpy as np

ds = zarr.open("dataset_suit.zarr", "r")
dof_names = list(ds.attrs["dof_names"])

# ── 0) 모델 버전 sanity check ───────────────────────────
print("n_dof =", ds["data/dof_pos"].shape[1], "| dof_names[5] =", dof_names[5])
# 'hip_flexion_l' → 23/27-DOF (인덱스 가정 정상)
# 'subtalar_angle_r' → 31-DOF 계열 (cable/hip_angles 필드 신뢰 금지)

# ── 1) 고관절 6-DOF 인덱스 자동 검출 ────────────────────
def hip_idx(names):
    keys = ("hip_flexion", "hip_adduction", "hip_rotation")
    r = [i for i,n in enumerate(names) if n.endswith("_r") and any(k in n for k in keys)]
    l = [i for i,n in enumerate(names) if n.endswith("_l") and any(k in n for k in keys)]
    return r + l

HIP = hip_idx(dof_names)
print("hip idx =", HIP, "→", [dof_names[i] for i in HIP])

# ── 2) 타깃 라벨 & 입력 피처 ────────────────────────────
y     = ds["data/dof_forces"][:, HIP]          # [N, 6]  고관절 토크 (N·m)
q     = ds["data/dof_pos"][:]                  # [N, n_dof]
qd    = ds["data/dof_vel"][:]                  # [N, n_dof]
act   = ds["data/actions"][:]                  # [N, n_act] raw 정책출력
cont  = ds["data/contacts"][:]                 # [N, n_body]

X = np.concatenate([q[:, HIP], qd[:, HIP], act[:, HIP], q, qd, cont], axis=-1)

# ── 3) 에피소드 경계 (시퀀스 윈도용) ────────────────────
ends = ds["meta/episode_ends"][:]            # 윈도가 이 경계를 넘지 않게
print("X", X.shape, "| y", y.shape, "| episodes", len(ends))
print("hip torque N·m range:", y.min(), y.max())
07

Toy 문제 설계 노트 — 한 가지 중요한 함정

design caveat

제어가 BUILT_IN_PD 이므로 토크는 거의 해석적입니다

skeleton 설정의 고관절 제어는 BUILT_IN_PD(stiffness=200, damping=20, effort_limit=600). 즉 시뮬레이터가 보고하는 고관절 토크는 1차적으로 다음 PD 법칙입니다:

τ_hip ≈ clip( 200·(target − q) − 20·q̇ , ±600 )

여기서 target 은 정책 action(아핀 스케일), q·q̇ 는 고관절 각도·각속도입니다. 따라서 입력에 action + 고관절 pos/vel 을 넣으면 IL 모델이 토크를 거의 그대로 복원합니다 — 좋은 sanity 베이스라인이지만 학습 난이도는 낮습니다.

더 의미 있는 과제를 원하면 action 을 빼고 운동학·접촉만으로 토크를 예측하게 하세요 (역동역학에 가까운 진짜 추론 문제가 됩니다). 어느 쪽을 택할지 먼저 정하는 게 핵심입니다.
